Retrobowl Sandlot

Retrobowl Sandlot is a fast, arcade-football prototype that blends classic sandlot vibes with a light franchise loop. You command a single offensive possession, sling mouse-aimed passes with an arcing power meter, and grind through a multi-week schedule that culminates in a two-round playoff chase. The experience is intentionally compact yet polished, focusing on crisp controls, readable UI, and dynamic receiver routes that feel alive from snap to whistle.

Core Features

  • Mouse-aimed Throwing – Hold the mouse button to build power, release to zip a pass anywhere on the field with vertical arc so defenders cannot block every attempt.
  • Receiver Route Visuals – SVG routes fade in before each snap, previewing WR, TE, and RB assignments for clear reads.
  • Animated Trenches – Offensive and defensive linemen battle to form a living pocket that affects pass timing.
  • Assignment-Based Defense – CPU defenders trail their assigned receiver rather than chasing the football blindly.
  • First-Down + LOS Markers – Persistent HUD elements show the chains and line of scrimmage for quick situational awareness.
  • Franchise Season Loop – An eight-week schedule escalates difficulty before the semifinal and championship rounds, with trophy rings for every title.

Controls

Action Input
Move QB / ball carrier Arrow Keys
Sprint Shift
Throw Click + hold to charge, release to fire toward the cursor
Reset Season R

Tip: Control automatically transfers to the receiver after a catch so you can keep running without extra input.

Season & Progression

  • Schedule Generation – Opponents are pulled from a curated pool with difficulty modifiers for the current season and week.
  • Playoffs – Finish with a winning record to unlock the Semifinal and Championship; hoist the trophy to add a ring and advance to the next season.
  • Dynamic Branding – HUD labels and the away endzone text update automatically to match every opponent.

Local Development

  1. Clone or download the repository.
  2. Open index.html in any modern browser.
  3. Play immediately—no build or tooling steps are required.

If you prefer live reload, serve the directory with a lightweight static server such as python -m http.server or npx serve.

Deployment Notes

  • The project is GitHub Pages-friendly: publish the root or main branch and the prototype will run as-is.
  • This HTML file mirrors the README so visitors can read the documentation directly from Pages.
  • To publish README.html, commit it to the branch used by Pages (commonly main) and enable GitHub Pages under Settings → Pages.
